## 用户旅程地图

### 店长每日补货流程（门店端）

```
登录 → 首页（看到待办） → 补货建议列表 → 查看详情 → 微调数量 → 确认下单 → 完成
```

### 区域经理查看数据（总部端）

```
登录 → 数据看板 → 选择区域 → 查看门店对比 → 发现问题 → 下发策略
```

### 管理员配置新租户（配置端）

```
登录 → 租户列表 → 创建租户 → 配置信息 → 分配配额 → 创建管理员账号 → 完成
```

## 设计原则

1. **移动优先** - 门店端 90% 场景在手机，大按钮、简洁布局
2. **行动导向** - 直接给出操作按钮，而非仅展示数据
3. **透明可解释** - AI 建议必须说明理由
4. **一致性** - 三端保持统一的视觉语言和交互模式
