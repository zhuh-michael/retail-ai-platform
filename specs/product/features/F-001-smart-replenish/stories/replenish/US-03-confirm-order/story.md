# 确认下单

**US ID**: US-03  
**所属分组**: replenish  
**状态**: draft  
**创建日期**: 2026-02-25

---

## User Story

**作为** 店长  
**我想要** 一键确认补货订单  
**以便于** 快速提交补货需求，无需重复操作

---

## 验收标准（AC）

1. **AC-1**: 店长可逐个确认单个商品的补货订单
2. **AC-2**: 支持批量确认（全选 → 一键提交）
3. **AC-3**: 确认前显示订单摘要（商品数量、预计金额、预计到货时间）
4. **AC-4**: 确认后显示成功提示和订单号
5. **AC-5**: 订单自动同步至现有采购/ERP 系统
6. **AC-6**: 支持撤销确认（在订单未处理前）

---

## 约束与假设

**约束**:
- 确认后必须记录操作日志（谁、何时、确认了什么）
- 必须记录 AI 建议 vs 实际确认的差异（用于 RLHF）
- 订单提交失败时必须有明确错误提示和重试机制

**假设**:
- 现有采购系统支持 API 对接
- 店长有权限提交补货订单

---

## UI 要点

- "确认下单"按钮醒目（主色调，大尺寸）
- 批量确认时显示进度反馈
- 成功后提供"查看订单详情"入口
